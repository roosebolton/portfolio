## HTML CSS JS DOM ##
### Beschrijving werkingsgebied HTML, CSS, Javascript, AJAX mbt webapplicaties in eigen woorden ###
Een webapplicatie biedt een user de mogelijkheid via een webpagina een programma te gebruiken. Zo'n
webpagina is dan de grafische interface. Webpagina’s worden doorgaans aan de gebruiker getoond via een browser.

### DOM ###
De DOM is een boomstructuur voor HTML, die door de browser wordt geparsed. Dit gebeurt van boven naar beneden. Alle knopen in de boomstructuur bestaan uit betekenisvolle tags, waarvan de HTML tag de root is in de DOM structuur.

Hieronder volgen beschrijvingen met korte voorbeelden van HTML, CSS, Javascript en Ajax die ik zelf schreef voor het cryptobankproject The Pirate Bank.

### HTML ###
HTML is applicatielaag taal die bestaat uit betekenisvolle tags. Aan de hand van de betekenis van de tags, de volgorde en nesting, weet de browser wat er moet worden weergegeven.
Tags kunnen worden verrijkt met extra informatie, die de weergave van de webpagina kunnen beïnvloeden, of andere talen, zoals CSS en Javascript, kunnen helpen bij het toekennen van uiterlijke kenmerken of gedrag aan de bewuste HTML elementen. Er kunnen bijvoorbeeld klassenamen of identiteiten aan worden toegekend, URL’s aan worden toegevoegd, of andere voor de specifieke tag relevante en parsable informatie.

```html
!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <!-- enzovoort -->
```

Een HTML pagina heeft een Head en een Body. In de Head worden hier algemene zaken, zoals de viewport en characterset bepaald in de meta tags. Meta tags horen binnen head tags thuis. In Link tags wordt informatie van
andere plekken binnengehaald, om de rest van de pagina in de zogenaamde body mee te kunnen laten werken. Hierboven zien we bijvoorbeeld dat ik CSS binnenhaal uit de eigen filestructuur op de eigen server, maar het
is net zo goed mogelijk om van bijvoorbeeld google fonts binnen te halen, of van bootstrap de library voor hun styling enz.

In de body valt de geneste structuur op. Ook vallen de tagnamen op en alle ander informatie binnen de tags. Door tags een klassenaam of id mee te geven, zijn ze per klasse of voor die ene id bereikbaar voor bijvoorbeeld styling met CSS of manipulatie door Javascript.

Styling kan ook in de tags zelf geregeld worden, maar het is netter op styling via CSS te regelen. Er zijn tal van betekenisvolle tags te zien. Bijna alle tags bestaan uit paren: En openingstag en een sluitingstag. De imagetag is hierop een uitzondering, die hoeft niet gesloten te worden.



### CSS ###
Met CSS kan het uiterlijk, de positionering en zelf het gedrag van HTML elementen worden beïnvloed. CSS kan HTML elementen vinden aan de hand van specifieke identifiers, zoals de id of class. Maar er zijn meer identifiers. Het geidentificeerde element kan met CSS allerlei transformaties ondergaan. CSS is een krachtig middel om een webpagina een eigen uiterlijk mee te geven. 
```css
.navbar-links {
    list-style: none;
    position: absolute;
    background-color: var(--navbar-bg-color);
    display: flex;
    flex-direction: column;
    align-items: center;
    left: 0;
    right: 0;
    margin: 1.4rem;
    border-radius: 5px;
    box-shadow: 0 0 20px rgba(0, 0, 0, 0.3);
}

#navbar-toggle[aria-expanded="true"] + #navbar-menu .navbar-links {
    padding: 1em;
}

@media screen and (min-width: 600px) {
    #navbar-toggle,
    #navbar-toggle[aria-expanded="true"] {
        display: none;
    }

    #navbar-menu,
    #navbar-toggle[aria-expanded="true"] #navbar-menu {
        visibility: visible;
        opacity: 1;
        position: static;
        display: block;
        height: 100%;
    }
```
In de CSS vallen een paar dingen op. Sommige elementen worden aangesproken via puntjes, andere via hekjes en er is ook een @media te zien.

De verschillende HTML elementen worden op basis van deze CSS bewerkt. Alle HTML elementen met een bepaalde class declaratie kunnen via een .{ CSS worden aangesproken. De elementen met een id worden via #{ benaderd. De @media annotatie zorgt na het toekennen van het soort media, en het setten van de parameter, een speciale behandeling voor elementen mogelijk is bij het bereiken van de door het instellen gestelde voorwaarde. Binnen de accolades van CSS staan de CSS styling variabelen en hun waarden.

Een voorbeeld: Wanneer het scherm minder breed wordt dan 600 pixels, wordt het toggle menu van de navbar getoond. De visibility wordt dan gezet
op visible. Bij .navbar-links zien we dat de background color wordt bepaald door een variabele. Deze variabele hebben we in een globale en te importeren CSS file gezet.

Verder zien we dat de positie van de navbar-links absolute is: relatief t.o.v. zijn ouder, of zonder ouder met de body meebewegend.

Er zijn ontzettend veel stylingsmogelijkheden met CSS. Belangrijk is te realiseren welke HTML je wil aanspreken en hoe precies, wat de ouders, sibling en kinder elementen zijn van het element of de elementen die je wilt stylen en hoe het positiegedrag is bepaald.


### Javascript ###
Javascript is een programmeertaal. Javascript wordt met name gebruikt om front end logica te schrijven, HTML elementen te manipuleren of creeren, maar zeker ook om met de backend te communiceren via (asynchrone calls)

```javascript
feather.replace()
const navbarToggle = navbar.querySelector("#navbar-toggle");
const navbarMenu = document.querySelector("#navbar-menu");
const navbarLinksContainer = navbarMenu.querySelector(".navbar-links");
let isNavbarExpanded = navbarToggle.getAttribute("aria-expanded") === "true";

const toggleNavbarVisibility = () => {
    isNavbarExpanded = !isNavbarExpanded;
    navbarToggle.setAttribute("aria-expanded", isNavbarExpanded);
};
```
In bovenstaande snippet staat wat Javascript code. Net als bij veel andere programmeertalen is het in javascript mogelijk om met variabelen en functies te werken. Wat we hier zien is bijvoorbeeld het
declareren en initialiseren van een variabele isNavbarExpanded.

Hier komt alles samen:
De HTML tag die als id navbar-toggle heeft wordt in de variabele navbarToggle opgeslagen. Vervolgens wordt in isNavbarExpanded gekeken of aria-expanded op true staat in deze navbar. De methode togleNavbarVisibility zorgt ervoor dat de waarde isNavbarExpanded het tegenovergestelde wordt van wat deze was, wat voor het togglegedrag zorgt. De CSS zorgt er bij het wijzigen voor dat de visibilty van het togglemenu wijzigt.



### Ajax ###
Asynchronous Javascript and XML is een werkwijze waarmee zonder de getoonde webpagina opnieuw te laden gegevens van de server of anders servers kunnen worden opgehaald of ernaar kunnen worden weggeschreven.
Ook kan ermee de inhoud van de huidige pagina worden aangepast zonder de pagina opnieuw te laden.

```javascript
/**
 * Fetches the assetRate information for all customers assets to extract current values from, to use with addToCurrentValue
 */
const setCurrentValues = () => {
    if (isEmpty(currentValues)) {
        let idList = Object.keys(idToCode)
        idList.forEach(id => {
            let getString = `${internalAssetRateEndPoint}${id}`
            fetch(getString, {
                method: 'GET',
                headers: {},
            })
                .then(response => {
                        return response.json()
                    }
                )
                .then(json => {
                    addToCurrentValue([id], json.current_price)
                })
                .catch((error) => {
                    console.error('Foutje', error);
                });
        })
    }
}
```
Hierboven is een javascript fetch te zien. Dit is een asynchrone javascript opdracht. Wat bovenstaand snippet laat zien is hoe, zonder de pagina opnieuw te laden, voor ieder element in een lijst met id's, voor iedere id
een fetch wordt uitgevoerd.

Deze fetch is een GET opdracht naar een in een variabele opgeslagen URL, waarachter de id wordt geplakt. In dit geval gaat het om een intern endpoint voor het ophalen van assetwaardes. Bij een succesvolle fetch
komt er een reactie, die in een json object wordt omgezet. Uit dit object wordt de huidige prijs gehaald en samen met de id toegevoegd aan een methode addToCurrenValue, die een object vult met id's als key. De pagina hoeft niet herladen te worden terwijl er wel allemaal verkeer is met het endpoint. Dit asynchrone functioneren is wat Ajax zo handig maakt.
